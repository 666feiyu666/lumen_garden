# 06 绘本美术资产与声音氛围接入 Harness Prompt

## 任务目标

在现有双模式可玩内容与 `P01-P05` 种植花谱交互已经稳定的基础上，将《萤光花园》的概念美术转换为游戏可实际加载、可缩放、可动画化的正式素材制作规格。

本轮讨论已确认：当前带有月夜花园、`10 x 10` 土格花圃、金色灯灵与右侧木框羊皮纸手记面板的概念图，不再只是探索参考，而是正式版本需要逐步逼近的**最终目标画面**。为了让目标画面能够真实互动，程序仍负责文字、数字、棋盘状态、按钮文案与交互反馈；生成式图片资产负责角色、植物、环境背景、土格/边框、装饰与空白 UI 皮肤。拆层是达成概念图的工程方法，不是另起一种风格方案。

音乐素材已经准备在 `assets/music/` 下，当前包含 `music_menu.mp3`、`music_puzzle.mp3` 与 `music_style.mp3`。首页及模式选择页使用 `music_menu.mp3`，教程、玩法与结算浮层使用 `music_puzzle.mp3`；`music_style.mp3` 作为制作参考不参与运行播放。

## 已确认设计决策

### 1. 绘本精灵风为当前生产基准

- 当前主方向为柔和的月夜绘本风：靛蓝背景、苔藓与石土、薄荷荧光花、暖金灯灵、木框与羊皮纸手记面板。
- 最终游戏界面以已选概念图的构图和完成度为目标：左侧大花圃、中央月夜路径纵深、右侧木框手记 HUD，以及围绕边缘生长的藤蔓、灯光与野花。
- 棋盘内精灵须简化轮廓和高光细节，以实际小尺寸可阅读为准，而不是追求概念大图中的全部纹理。
- 菜单与背景可以保留更精致的绘本质感；棋盘中的灯灵、花芽和状态提示需要更克制、更清楚。

### 2. 像素风保留为可行备选，不作为当前并行生产线

- 像素风可能改善棋盘小尺寸辨识和动画一致性，因此不得被排除。
- 但像素风会显著改变月夜绘本氛围，也不能天然解决生成素材的比例、配色与帧间一致问题。
- 仅当绘本精灵包在真实棋盘尺寸下出现持续不可读、遮挡严重或资产难以统一的问题时，才制作一组同规模像素对照样本。
- 像素方案的比较对象应为同一套最小游戏资产：棋盘灯灵、花苞、盛开花与衰退花；不先重做整幅背景或完整 UI。

### 3. 灯灵具有叙事形态与操作形态

- 初始界面、引导页面与结算可使用正视灯灵：脸部与卷曲火苗清晰，负责建立角色亲和力。
- 实际花圃操作使用高角度俯视灯灵：身体更紧凑，负责表达当前格位置，并避免覆盖花芽状态。
- 两种形态必须共享暖金色内光、水滴轮廓、顶端卷曲、简洁眼睛与伴随星光，使玩家明确识别为同一角色。
- 当灯灵与新种花芽位于同一格时，花芽仍必须可见；位置金环和轻微悬浮偏移可由程序表现。

### 4. 生成资产使用透明 PNG，而不是自动转换 SVG

- ChatGPT-image 生成的角色、植物和纹理属于带渐变与柔光的位图资产，最终游戏格式使用透明背景 `PNG/RGBA`。
- 可先以纯色抠图底生成母版：暖金灯灵使用纯绿 `#00ff00` 背景；薄荷色植物使用纯洋红 `#ff00ff` 背景。
- 原始纯色底图片可保留为后续生成变体的参考母版；游戏运行只加载去底后的透明 PNG。
- `SVG` 仅考虑用于规则边框、简单图标或可确定性绘制的 UI 装饰，不用于由图片模型生成的灯灵和花朵插画精灵。
- 大范围外发光不烘焙进角色边缘；呼吸光、定位金环和部分粒子优先由程序绘制或作为独立透明素材提供。

## 资产生产顺序

### A. 第一批：棋盘可玩性精灵包

它们用于直接替换或叠加现有程序绘制标记，在 `P02 BLINKER` 等真实关卡里验证阅读性、播种反馈和循环演示效果。灯灵正视与俯视素材已经进入工作区；花芽状态与后续界面素材依据 [美术概念图 Prompts](../docs/art-prompts.md) 中的生产提示继续生成。

| 素材 | 用途 | 输出建议 | 优先级 |
| --- | --- | --- | --- |
| `lantern_portrait_idle.png` | 菜单/引导中的正视灯灵母版 | 透明 PNG，高分辨率保留 | 已准备 |
| `lantern_board_idle.png` | 花圃中的高角度俯视灯灵 | 透明 PNG，缩至约 `40-56 px` 仍清楚 | 已准备 |
| `bud_planted.png` | 播种后与当前存活花芽 | 透明 PNG，缩至约 `32-44 px` | 必须 |
| `bloom_open.png` | 盛开或稳定状态 | 透明 PNG，与花苞明显区分 | 必须 |
| `bloom_fading.png` | 将消散/失败反馈 | 透明 PNG，花瓣松散但仍可识别 | 必须 |
| `petal.png` | 消散与胜利的轻量装饰 | 小型透明 PNG | 可随后补充 |
| `sparkle_gold.png` / `sparkle_mint.png` | 灯灵与盛开的粒子点缀 | 小型透明 PNG，或改由程序绘制 | 可随后补充 |

首批不需要生成逐帧角色动画。灯灵漂浮、呼吸光、移动残光，花朵淡入淡出与花瓣漂落，优先利用单张精灵配合程序动画实现。

### B. 第二批：花圃表面与读图支撑

棋盘表面与边框需要严格服从最终概念图：深色土格、石/木混合边框、藤蔓覆盖外围。棋盘精灵比大背景更优先，因为玩家持续观察的是格子与花朵状态。

| 素材 | 用途 | 约束 |
| --- | --- | --- |
| `soil_tile_empty.png` | 空土格基础皮肤 | 低对比、可平铺，不包含花或高亮 |
| `soil_tile_allowed.png` 或程序高亮 | 可播种/可移动反馈 | 优先以程序边框叠加，避免生成过多状态贴图 |
| `board_frame.png` 或角部装饰 | 花圃外围木框/石框 | 不改变可点击格尺寸，不覆盖边缘单元 |

不得生成一张已经画死花朵位置的完整棋盘作为正式素材。棋盘状态由程序动态摆放精灵。

### C. 第三批：HUD 面板与按钮皮肤

选项框和右侧手记面板应在棋盘视觉成立后制作，因为其颜色与厚重程度需要服从棋盘可读性。

| 素材 | 用途 | 约束 |
| --- | --- | --- |
| `journal_panel_blank.png` | 右侧信息面板底板 | 完全无文字、无数字，中心留空 |
| `button_idle.png` | 菜单或操作按钮底板 | 无文字，适合程序叠字 |
| `button_hover.png` / `button_selected.png` | 悬停和当前选择状态 | 与普通按钮尺寸、内边距一致 |
| `card_blank.png` | 关卡卡片/模式入口 | 无文字与固定徽章内容 |

如果后续需要窗口伸缩，面板和按钮应设计为可九宫格拉伸的边框结构；不要把完整菜单选项文字生成在图内。

### D. 第四批：无 UI 大背景

大背景与木框手记面板共同决定最终概念图的世界质感，已纳入正式素材目标。生产时仍按拆层导出，确保程序可以动态绘制棋盘状态与中文 HUD。

需要的背景类型：

| 素材 | 场景 | 约束 |
| --- | --- | --- |
| `menu_background.png` | 初始界面 | `16:9`，为标题、按钮和正视灯灵留出安静区域 |
| `gameplay_background.png` | 花圃/手记界面 | `16:9`，花圃与 HUD 所在区域低对比，不绘制固定 UI |
| `result_overlay.png` 或程序光效 | 胜利/成谱展示 | 保留棋盘可读性，不以整块插画覆盖花谱 |

背景不得包含可读文字、按钮、数字、固定花圃图案或需要交互的角色。

### E. 第五批：音乐与音效接入

首页音乐已有候选/来源文件；运行时继续使用以下资源入口补入最小音频集合：

```text
assets/music/music_menu.mp3
assets/music/music_puzzle.mp3
assets/sfx/move.wav
assets/sfx/grow.wav
assets/sfx/reject.wav
assets/sfx/win.wav
assets/sfx/lose.wav
assets/sfx/select.wav
```

已准备两首安静、循环不疲劳的月夜花园氛围音乐；首页及模式选择页循环播放 `music_menu.mp3`，教程、玩法与结算浮层循环播放 `music_puzzle.mp3`。音效首先保障移动、播种/生长、拒绝和成功等操作反馈可辨认，不以大量环境声盖过思考节奏。

## 首轮生成与验证方案

### 生成顺序

1. 已准备正视透明灯灵 `lantern_portrait_idle.png` 与俯视透明灯灵 `lantern_board_idle.png`，作为同一角色的两种显示形态。
2. 依据最终概念图生成 `bud_planted.png`、`bloom_open.png` 与 `bloom_fading.png`。
3. 依据最终概念图生成土格、花圃边框、右侧空白手记面板、按钮皮肤和无文字背景板。
4. 后续资源接入时使用真实 `10 x 10` 正式花圃验证：移动、播种、灯灵与花芽叠放、生长循环与成功展示。
5. 首页双曲轮播已接入；后续再补充玩法音乐与交互音效。

### 游戏尺寸验收

- 正视灯灵在菜单尺寸下应有角色魅力，且能作为后续俯视版本的造型参考。
- 棋盘灯灵缩小至实际格内尺寸后，仍能与绿色花芽一眼区分。
- 花苞、盛开、衰退三状态缩小后仍能仅凭轮廓与亮度区分，而不是依赖细小纹理。
- 灯灵位于已种下花芽的格子上时，二者同时可见，不能再次产生“按了 `P` 但看不到播种”的误解。
- 透明 PNG 在深靛蓝背景上没有明显绿边或洋红边；柔光不污染格子边界。
- HUD 和未来背景不会抢过交互高亮、文本与目标状态的视觉优先级。

## 本 Harness 不包含的实现工作

- 不把概念稿中的包含动态状态的整张界面直接当作可交互 UI；目标视觉通过分层素材组合实现。
- 不把图片模型生成的角色精灵自动追踪转换成 `SVG`。
- 不同时生产完整像素版资源；仅保留在绘本精灵验证失败或需要明确对照时启动的备选路线。
- 不在美术生产规格中更改游戏规则、关卡内容、输入方式或状态机。
- 不将 `music_style.mp3` 制作参考音频作为游戏曲目播放。
