# 05 播种确认与花谱循环展示 Harness Prompt

## 任务目标

在不继续增加关卡内容的前提下，打磨已经完成的 `P01-P05` 种植花谱第一册：让玩家明确知道自己何时在播种、何时在放手验证，并让成功形成的动态花谱能够被持续观看。

本任务来自对当前试玩闭环的交互评审。现有实现已经具备播种、自动验证、暂停和单步，但存在两类关键可读性问题：

- 播种阶段中，`Space` 直接种下一枚花种，鼠标点击灯灵也会隐式播种；它们与自由演化模式中“停留/推进”的习惯冲突，容易产生误操作。
- 播种阶段中，`Enter` 可以在仍有花种未布置时直接启动一次注定失败的验证，未把“花苗尚未种完”作为可修正的准备状态反馈给玩家。
- 生长验证达到目标代数后立即以结果遮罩结束播放；周期和移动图案刚刚展现行为就停止，玩家无法把它当作一张活着的花谱继续观看。

`05` 的方向是交互确认与成谱后的观察反馈，不是新增谜题、改变经典规则或全面重做 UI。

## 已确认设计决定

### 1. 播种必须成为明确动作

- 仅在种植花谱的**播种阶段**调整键位；花园谜题和月夜入门中的 `Space = 停留并生长` 保持不变。
- 键盘播种改为 `P`，与界面中的“播种”动作建立清楚对应；`X` / `Backspace` 继续用于撤回玩家本轮播下的花种。
- 鼠标玩家仍能完成全流程：明确标注的“播种”按钮负责播种。点击灯灵所在格不再无提示地消费花种，可给出“按 P 或点击播种按钮”的提示。
- 播种阶段的 HUD 与提示文案必须持续显示关键控制：移动、`P` 播种、`X` 撤回、花种齐备后 `Space` / `Enter` 开始生长。

### 2. 未完成播种时不能启动验证

- `PlantingState` 的规则层应拒绝在 `seeds_left > 0` 时开始生长，而不只在界面层隐藏或拦截按钮，确保键盘、鼠标与后续入口行为一致。
- 播种阶段按 `Space` 或 `Enter` 时，若仍有剩余花种，保持当前布局并提示例如“还有 1 枚花种未种下，完成播种后再开始生长”。
- `Space` 不再承担播种，但在花种已用完时与 `Enter` 共同作为启动验证的冗余确认键；鼠标在花圃内仅用于点击相邻格移动，不由点击灯灵触发确认。
- 无效开始、误按 `Space` 与无效播种都不得消耗花种、改变花芽集合或推进代数。

### 3. 判题完成后进入成谱演示

- 验证阶段仍以当前纯 `B3/S23` 演化和目标代数判定成功/失败；不得为显示效果改写题目答案或允许灯灵参与验证。
- 失败结果保持清楚且可立即重新播种，不自动循环错误图案。
- 成功结果不再用永久遮罩阻挡花圃。右侧札记或轻量结果标记说明“成谱”原因，同时花圃进入可观看的**成谱演示**状态。
- `P02-P05` 成谱后默认继续播放其已验证动态片段，并支持暂停/继续与从片段起点重播；`P01 Block` 可持续显示稳定盛放与呼吸光效，并明确其“不变”即验证结果。
- 成谱演示是显示/复盘层，不是额外判题轮次：保留已完成状态、胜利文案和正确解的记录，不因后续展示帧变化而反转胜负。

### 4. 有限花圃中的循环展示边界

- `Glider` 与 `LWSS` 不能在现有有限 `10 x 10` 花圃上被误称为可永久自然飞行；继续向边缘演化会改变甚至摧毁图案。
- 因此，成谱演示应循环重播题目已经安全验证的动画片段，例如从锁定的正确起始布局播放至目标代数，再以清楚而柔和的过渡回到片段起点。
- 循环片段应复用纯规则生成的帧或从已锁定正确布局重新演化，不以手写像素动画替代规则结果。
- UI 可将这一状态命名为“成谱演示”或“循环回看”，避免让玩家误以为循环重置本身是康威规则的一部分。

## 交付范围

### M1：播种输入与即时反馈

- 在 `Lumen_Garden/app.py` 中调整种植场景的键盘及鼠标映射：`P` 播种，`X` / `Backspace` 撤回，播种齐备后 `Space` / `Enter` 开始验证；花圃内鼠标点击只移动或提示；保留按钮、重置、返回。
- 更新种植界面动作说明、按钮反馈和消息提示，使当前阶段、剩余花种和下一项有效动作无需猜测。
- 检查种植菜单及结果后的重试入口，避免 `P` 与主菜单进入种植花谱的快捷键产生场景内歧义。

### M2：开始生长的规则保护

- 在 `PlantingState.start_growth()` 加入剩余花种检查，并提供适合界面展示的拒绝原因。
- 保持正确布局、错误但已用完花种的布局都可进入验证：前者能成功，后者应按现有规则失败；只阻止“尚未完成播种”的提前提交。
- 不改变 `GameState`、教程或自由演化模式的移动和等待规则。

### M3：成功后的成谱演示循环

- 为种植花谱成功后增加独立于判题的演示生命周期，保存或可重建被验证的初始布局与演示片段。
- 使成功时的花圃仍可阅读，不被大面积结果遮罩长期盖住；保留清楚的成功原因与重置/返回操作。
- 支持动态花谱的自动循环、暂停/继续，以及至少一种明确的重新从片段开头观看方式。
- 对 `Block`、周期图案与移动图案分别给出合理反馈；尤其保证 `Glider` / `LWSS` 不因演示继续推进至边缘而呈现“成功后自行毁坏”的假象。

### M4：玩家文档与测试

- 执行实现时同步 `README.md`、说明书、游戏流程及种植花谱设计文档中的实际键位、阶段说明和成谱演示状态。
- 将“播种是否应独立按键”从待试玩问题改写为本轮采用的决定，并继续保留尚未完成的难度试玩、收藏/存档等问题。
- 补充规则与 dummy SDL UI 测试，覆盖输入防误触和成功后循环展示的关键路径。

## 测试门槛

- 既有花园谜题、教程、经典图案规则和 `P01-P05` 成功/失败验证测试继续通过。
- 状态测试验证：剩余花种大于零时 `start_growth()` 被拒绝，布局、余额、阶段和代数均保持不变。
- UI 测试验证：播种阶段 `Space` 不会种花，`P` 会在合法位置消耗花种；未种完时 `Space` / `Enter` 不会进入生长验证并显示提醒，种完后两者均可启动验证。
- UI 测试验证：使用键盘和鼠标按钮均可完成至少一则种植题，不依赖点击灯灵进行隐式播种。
- 播放测试验证：正确完成动态花谱后仍处于已成功状态，成谱演示可以继续更新可见帧并暂停/重播，不会改变成功判定。
- 播放测试验证：`Glider` 与 `LWSS` 的演示循环以已验证片段为边界，不因继续自动推进撞上花圃边缘。

## 不在本轮范围

- 不新增 `P06` 或新的经典图案，不在没有试玩数据的情况下重排五关难度。
- 不改变两种主玩法的规则边界：自由演化仍含灯灵邻居，种植验证仍为纯规则。
- 不在本轮实现完成进度存档、花谱收藏册、独立自由观察模式或正式美术/音乐资产。
- 不用无限棋盘、环绕边界或额外规则来掩盖移动花谱的有限棋盘展示问题。

## 验收标准

- 玩家在播种阶段不会因为沿用 `Space` 的既有习惯而意外播种；键盘与鼠标都有明确可见的播种动作。
- 尚有花种未布置时，`Space` 与 `Enter` 都会反馈未完成准备，且不会改变谜面或开始失败演出；完成后任一键均可确认开始。
- 玩家正确播种后仍可一键开始自动验证，并照常得到基于经典行为的成功或失败判定。
- 任一成功花谱完成后，玩家能够继续观看其稳定、周期或移动表现；动态成谱不会在刚被证明后立刻被遮挡和停掉。
- 展示循环不歪曲有限棋盘与纯规则验证的事实，也不破坏现有测试和双模式内容。
