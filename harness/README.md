# Harness Prompts

本目录存放可直接交给编码 agent 的阶段性执行提示词，与 `docs/` 中的游戏设计、流程和说明文档分开管理。

已执行或正在执行的提示词按阶段归档：

```text
harness/
  phase-1/    第一阶段原型、菜单、花谱、美术与音乐接入
  phase-2/    第一章荧光森林与后续章节开发
```

## 维护规则

- 新阶段提示词放入对应 `phase-*` 目录，不覆盖已经执行过的提示词。
- 提示词应记录任务目标、实现边界、测试要求和实际开发记录。
- 当实际实现与原提示词不同步时，应在同一提示词末尾追加“实际开发记录”，保留决策轨迹。
