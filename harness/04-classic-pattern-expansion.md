# 04 经典动画花谱扩展 Harness Prompt

## 任务目标

在统一 `10 x 10` 的种植花谱上，将康威生命游戏中适合有限花圃短程观察的经典动画移植为四则新关卡，使第一册从静物推进到周期与位移行为。

## 设计决定

- 延续“先播种，后验证”：灯灵仅在播种阶段布置有限花种，生长验证阶段使用纯 `B3/S23`。
- 延续 `03` 建立的自动播放、暂停和单步验证，不新增中途干预规则。
- `P02 BLINKER / 摇曳花枝`：周期 2，补一枚花种，第 2 代回到原相位。
- `P03 BEACON / 双灯花台`：周期 2，补一枚花种；较宽轮廓作为飞船前的阅读过渡。
- `P04 GLIDER / 流光种`：补一枚花种，第 4 代斜移 `(1, 1)`。
- `P05 LWSS / 晨风舟`：补两枚花种，第 4 代水平移动 `(2, 0)`；以新的运动方向避免末关重复 Glider。
- 大型或长寿命图案暂不采用：它们在 `10 x 10` 的有限花圃内难以提供短而清楚的验证过程。

## 交付范围

### M1：经典图案与关卡数据

- 在 `patterns.py` 中加入 `Beacon` 与 `LWSS` 标准形状，并保留 `Block`、`Blinker`、`Glider`。
- 定义 `P02-P05` 的预置花芽、花种预算、初态目标、验证代数与演化后目标位置。
- 所有题目继续使用 `PlantingState` 与既有纯规则演化，不为某个图案硬编码胜利捷径。

### M2：选择页与交互文案

- 让种植花谱选择页在一屏展示并可鼠标点击 `P01-P05`。
- 验证信息使用适合静物、周期与位移图案的通用表述。
- 播种提示明确有限花种，而不把所有关卡描述成教学补种。

### M3：测试与文档

- 规则测试验证 `Beacon` 周期 2 与 `LWSS` 四代横移。
- 状态测试验证五关按目标播种后都可通过实际演化结算。
- UI 烟雾测试覆盖 P05 的鼠标入口。
- 同步设计文档、流程、说明书与 README 的当前内容说明。

## 验收标准

- 种植菜单展示并可进入五则花谱。
- `P01-P05` 的正确初态均可通过自动/单步演化得到胜利，错误初态仍失败。
- `Glider` 与 `LWSS` 在碰到有限花圃边缘前完成其位移验证。
- 文档不再将 P02-P05 描述为未实现，也不将尚未试玩调优的难度梯度描述为定稿。
