# Phase 2 Harness Prompt 02/03: 五房间森林与荧光花园 Demo

## 任务目标

在 `Phase 2 Harness Prompt 01` 的三房间剧情闭环基础上，补齐第一章可试玩 demo 的主要缺口：

- 将荧光森林扩展为五个房间，形成更完整的机制梯度。
- 加入背包与图鉴，让森林探索获得可回收的资源和知识。
- 将“荧光花园”从占位页改为可交互的六块花圃 hub。
- 将森林 UI 从花园谜题风格中分离，改成森林迷宫与短时对话框结构。

## 五房间森林

当前第一章主线采用 `00 -> 01 -> 02 -> 03 -> 04` 的线性结构：

| 房间 | 主题 | 当前功能 |
| ---: | --- | --- |
| 00 | 引路空庭 | 无植物，熟悉上下左右移动并到达出口。 |
| 01 | 普通植物房 | 普通植物随行动演化；房间内放置一枚普通植物种子，拾取后进入背包。 |
| 02 | 静物图案房 | `Block / 四叶眠床` 被 4x4 石头外框保护，靠近后记录到图鉴。 |
| 03 | 机关植物房 | 拾取 `Glider / 流光种` 缺失种子，站到缺口按 `P` 补回并启动。 |
| 04 | 脉冲门廊 | `Blinker / 摇曳花枝` 在两个按钮之间相位交替；出口随按钮上是否存在植物红绿切换。 |

## 背包与图鉴

- 背包当前记录两类资源：普通植物种子与流光种。
- 图鉴当前记录三类可复现图案：`BLOCK`、`GLIDER`、`BLINKER`。
- 森林右上角提供“背包”和“图鉴”按钮，点击后以短时对话框显示当前内容。
- `P` 在无可补种机关时作为背包提示；在机关房中用于补回缺失种子并启动机关植物。

## 森林 UI

- 棋盘使用森林专属布局，居中并放大，不再复用花园谜题的右侧说明栏。
- 左上角显示当前房间与生命代数。
- 右上角显示背包、图鉴、重试、返回按钮。
- 引导性文字改为底部 Galgame 风格对话框。
- 对话框最长显示 5 秒，玩家点击对话框本体可以立即收起。

## 荧光花园 Hub

荧光花园入口当前已从“正在开发”占位页替换为六块可点击花圃：

- 每块花圃使用 `puzzle_board` 作为小棋盘底图。
- 点击空花圃会根据背包与图鉴尝试自动选择可种植内容。
- 种植成功后消耗相应背包资源，并在该花圃上绘制植物/图案。
- 点击已种植花圃会显示该花圃当前种下的内容。

当前种植门槛：

| 内容 | 门槛 |
| --- | --- |
| 普通植物 | 1 枚普通植物种子 |
| 四叶眠床 | 图鉴记录 `BLOCK`，且拥有 4 枚普通植物种子 |
| 摇曳花枝 | 图鉴记录 `BLINKER`，且拥有 3 枚普通植物种子 |
| 流光种 | 图鉴记录 `GLIDER`，且拥有 1 枚流光种 |

第一版暂未做“点击花圃 -> 选择图鉴条目 -> 确认种植”的完整选择面板；当前使用自动选择可种内容作为交互原型。

## 测试记录

新增和更新的 smoke tests 覆盖：

- 主线进入森林后从第 0 房间开始。
- Phase 2 主线包含 0-4 五个编号房间。
- 森林使用独立迷宫棋盘，不调用花园谜题底图。
- 森林对话框最多 5 秒，并可点击收起。
- 第 01 房间可拾取普通植物种子。
- `Block` 靠近后写入图鉴。
- `Blinker` 双按钮会让出口红绿交替。
- 荧光花园入口打开六花圃 hub。
- 花圃可消耗普通种子种下普通植物。
- 图鉴与足量种子满足时可种下 `Block`。
- 花园 hub 对话框可点击收起。

## 后续建议

- 将荧光花园的自动种植升级为图鉴选择面板。
- 增加更多普通植物种子来源，否则 `Block` / `Blinker` 的门槛只能通过测试或后续关卡资源满足。
- 为花园 hub 增加持久化存档，保留已种植花圃、背包和图鉴。
- 继续替换森林植物、种子、按钮、门与对话框美术，使其更接近最终绘本风。
