# 《萤光花园》戏剧方向决策

## 决策结论

项目原型曾以“变异细胞 / 生物实验”为包装。试玩与概念图检视后，确认培养皿、变异和半写实细胞容易产生阴冷或令人不适的联想，与希望建立的轻巧解谜体验不符。

当前选择：

| 方向 | 定位 | 状态 |
| --- | --- | --- |
| **萤光花园 / Lumen Garden** | 金色灯灵引导月夜花芽盛放的温暖解谜游戏 | 当前实现主方案 |
| **星图守望 / Starwatch** | 引导星移动并令星点汇成稳定星图的安静解谜游戏 | 保留为候选主题/未来变体 |
| 变异细胞 / Mutant Cell | 实验室中调控细胞群落 | 停止作为对外视觉与叙事方向 |

底层康威规则可以继续使用；正式第一册已改为统一 `10 x 10` 的新谜题布局，变化同时覆盖关卡节奏、名称、戏剧动机、界面语言、美术与音频表达。

## 主方案：萤光花园

### 一句话体验

玩家带领一枚温暖的金色灯灵，在黎明到来前穿行月夜花圃，通过自己的光照位置，让恰好数量的花芽盛放。

### 戏剧语汇转换

| 原型概念 | 萤光花园表达 |
| --- | --- |
| 红色不死玩家细胞 | 金色灯灵 |
| 普通细胞 | 发光花芽 |
| 出生 | 萌芽 / 盛放 |
| 死亡 | 熄灭 / 凋落 |
| 演化 | 生长一轮 |
| 培养皿 | 月夜花圃 |
| 关卡 | 谜题 |
| 目标种群 | 盛开目标 |
| 回合限制 | 黎明倒计时 / 可用步数 |
| 实验成功 | 花园盛放 / 谜题解开 |
| 实验失败 | 月夜散场 / 未能成景 |
| 规则训练 | 月夜入门 |

### 为什么它适合规则

- 花芽之间的“相伴而活、孤单而灭、三束光催生新芽”能够温和解释康威规则。
- 灯灵作为活邻居的作用直观：它提供一束额外微光，但不算作花芽数量。
- 有限步数可以自然解释为黎明前的短暂夜色，而不必诉诸实验资源或惩罚。
- 谜题目标强调营造图景，不需要围绕难度等级或闯关压力建立包装。

### 谜题呈现原则

- 内容选择不是难度阶梯，而是两册月夜花谱：引导花园与种植花谱。
- 谜题可有名称和情绪，例如“初光”“新芽”“合鸣”“月圃”“黎明”。
- 现有五个布局暂作为第一册谜题，后续不必按简单/困难命名。
- 评价重点可由“通关”逐步扩展为步数、不同解法、最终花形美感。

## 备选方案：星图守望

### 核心设定

玩家控制一颗金色引导星，在深蓝天幕的星格之间移动；星点会依邻近光芒聚合或淡去，玩家需在夜尽前完成一幅恰好的星图。

### 保留理由

- 同样摆脱生物恐怖感，且抽象清洁，适合程序绘制。
- 如未来需要极简主题、夜空扩展章节或换肤模式，星图可复用同一规则系统。
- 与萤光花园共享“微光、夜晚、引导、平衡”的情绪内核，美术探索不会完全浪费。

### 暂不作为当前实现的原因

花园比星图更容易表达“生长、盛放、凋落”的反馈，也更适合首次教学；因此本轮优先实现萤光花园。

## 当前制作约束

- 游戏中所有玩家可见文本应使用花园谜题语汇。
- 不再生成培养皿、病菌、组织、医疗监测或生物实验方向的正式概念图。
- 概念图可提供背景和构图，文字、数值和关键提示仍由程序绘制。
- Python 包与玩法数据已迁移为 `Lumen_Garden`、`Puzzle` 与 `blooms` 语义；工作区根目录路径暂保留，避免打断当前 IDE 工作区。
