# 房间与机制规范

## 1. 房间定义

Phase 2 的基本关卡单位是“房间棋盘”。每个房间是一张独立棋盘，包含入口、出口、普通植物、机关植物和局部目标。玩家走到房间出口后进入下一房间，第一章先由三个线性房间组成。

房间不只是不同背景下的同一花圃。一个合格房间应回答：

- 玩家从哪里进入。
- 玩家要怎样离开。
- 哪些格子能走、能生长、会失败或会阻挡。
- 当前房间教授或复用哪一个机制。
- 玩家如何知道机关已经启动。

## 2. 植物分类

荧光森林中的植物分为普通植物与机关植物。两者底层都遵循康威生命游戏规则，但它们在关卡设计中的身份不同。

| 类型 | 示例 | 规则身份 | 关卡身份 |
| --- | --- | --- | --- |
| 普通植物 | 花、蘑菇、荧光芽 | 每次灯灵行动都会进行生死判断 | 道路阻挡、出生死亡教学、森林氛围 |
| 机关植物 | `Glider`、大炮、未来飞船类植物 | 未启动前休眠；启动后按生命规则演化 | 触发机关、开门、发射、运输 |

设计上要避免让“机关植物”看起来像脱离康威规则的魔法按钮。它们应该仍然以植物格构成，只是房间机关会识别某些图案行为。未启动前的机关植物是被遗迹锁住的休眠形状；启动后，它们才成为会随回合变化的生命图案。

## 3. 地形词汇

| 地形 | 灯灵 | 植物 | 机关 | 用途 |
| --- | --- | --- | --- | --- |
| 土格 | 可行走 | 可生长 | 可放置 | 常规解谜空间 |
| 黑水 / 虚空 | 走入失败 | 飞船可跨越 | 通常不可放置 | 后续章节障碍，第一章可暂不实装 |
| 石壁 | 阻挡 | 撞击熄灭 | 不可放置 | 划分房间形状和防止图案无限扩散 |
| 遗迹底座 | 可站立或不可站立，按房间定义 | 通常不可自然生长 | 可安装机关元素 | 拼接机关的位置提示 |
| 出口门 | 条件开启后可通过 | 不生长 | 受机关控制 | 房间完成条件 |

第一章不要让地形规则过多。第一版优先使用土格、石壁、门、底座；黑水和载具可以留到后续章节。

## 4. 回合与代数

Phase 2 不再使用一条简单的“移动是否演化”规则，而是按植物类型区分。普通植物始终随灯灵行动演化；机关植物在未启动前不会演化，启动后才随灯灵行动或等待推进。

| 概念 | 含义 |
| --- | --- |
| 行动回合 | 玩家移动、拾取、放置、旋转、进入载具或撤销 |
| 生命代数 | 已激活的植物集合按 `B3/S23` 同步演化一次 |

推荐输入语义：

| 操作 | 普通植物 | 未启动机关植物 | 已启动机关植物 | 说明 |
| --- | --- | --- | --- | --- |
| 移动 | 演化一代 | 不演化 | 演化一代 | 房间一沿用现有引导体验；启动后的机关植物进入同一节奏 |
| 等待 | 演化一代 | 不演化 | 演化一代 | 用于观察和推进图案 |
| 拾取/放置种子 | 演化一代或按房间规则暂停 | 不演化 | 演化一代 | 第一章可为了教学清晰，让房间二放置阶段暂停普通植物 |
| 启动机关植物 | 演化一代或从下一回合开始 | 从休眠转为已启动 | 演化一代或从下一回合开始 | 具体时机需在关卡规格中写明 |
| 撤销 | 回退 | 回退 | 回退 | 支持精确解谜 |

在混合房中，普通植物与已启动机关植物可以同一回合一起演化；未启动机关植物保持形状不变。这样玩家会理解：机关植物不是不遵守生命游戏，而是在启动前尚未“通电”。

## 5. 普通植物

普通植物是房间的自然规则层，默认随每次灯灵移动或等待按有限棋盘上的 `B3/S23` 演化：

- 活花周围有 2 或 3 个邻居则延续。
- 活花周围少于 2 或多于 3 个邻居则熄灭。
- 空土格周围恰好 3 个邻居则萌芽。
- 石壁、黑水和机关底座是否允许生长由房间数据决定，第一章默认只允许土格生长。

普通植物可以处于两种来源：

| 来源 | 用途 |
| --- | --- |
| 房间初始植物 | 构成道路阻挡、环境节奏和教学谜面 |
| 灯灵影响后生成的植物 | 让玩家理解灯灵也会参与邻居判断 |

房间一设计重点：

- 出口或道路一开始被普通植物挡住。
- 玩家从棋盘下部出发。
- 玩家在前进、绕行或等待时发现植物会生长与死亡。
- 灯灵靠近时也会构成生长或死亡条件。
- 这个房间可以沿用现有引导模式的设计，不需要背包或机关拾取。

## 6. 机关植物与机关元素

机关植物是能被房间机关识别的生命图案。机关元素则是玩家可搬运、拼接或安装的对象，用来补全机关植物或连接触发目标。机关植物在未启动前保持休眠，不因灯灵移动而生长、死亡或位移；启动后，它们按普通生命游戏规则随行动回合推进。

第一章建议只保留一种机关植物和一种机关元素：

| 元素 | 玩家动作 | 启动结果 |
| --- | --- | --- |
| 残缺 `Glider` | 观察其缺口和方向 | 等待被补全，未启动前不演化 |
| `Glider` 种子 | 拾取并种回缺口，或站到缺口附近播种 | 补齐并启动后演化，触碰机关 |

机关元素设计原则：

- 一次只让灯灵携带一个元素。
- 元素应有清楚外观，避免和普通植物混淆。
- 放置合法性由底座、高亮和拒绝反馈说明。
- 拼接成功后不要立刻自动解谜，最好让玩家明确启动，随后通过移动或等待观察结果。

## 7. 滑翔机：远程触发

`Glider / 流光种` 是第一章的远程触发机关植物。未启动前，它保持残缺或完整的休眠形状；启动后，它按生命游戏规则演化，每四代斜向平移一格。第一版可以让它触碰一个简化机关形状来开门，正式版再替换为月光接收器。

房间语法：

```text
残缺流光种 + 一枚可补回的种子 + 发射方向 + 机关形状/接收器 + 被锁住的门
```

关键设计点：

- 接收器或占位机关必须位于玩家走不到、但 `Glider` 可以抵达的位置。
- 玩家应能预判方向，早期房间可用浅色轨迹、缺口高亮或目标形状提示。
- `Glider` 撞到石壁或门时熄灭，撞到接收器则触发机关。
- 成功触发后，门、桥或锁应有明确动画和声音反馈。

难度梯度：

| 阶段 | 谜题要求 |
| --- | --- |
| 入门 | 已经摆好 `Glider`，玩家只需等待四代 |
| 基础 | 玩家补一枚花种完成初态 |
| 进阶 | 玩家需要选择发射方向或启动时机 |
| 时间差 | `Glider` 飞行期间，灯灵要赶往另一个机关配合 |

## 8. 晨风舟：载具渡水

`LWSS / 晨风舟` 是后续章节的载具工具。它每四代水平移动两格。灯灵站到阵眼后进入搭乘状态，收敛自身光芒，不再作为植物邻居干扰图案。

当前第一章三房间版本暂不实装 `LWSS`，只在设计文档中保留为后续扩展方向。

房间语法：

```text
黑水断路 + 残缺晨风舟 + 阵眼核心 + 对岸出口
```

搭乘规则建议：

- 灯灵必须站在飞船阵眼格并执行“搭乘”或“等待”确认。
- 搭乘后灯灵不计入生命游戏邻居。
- 每次等待推进一代；当飞船完成位移，灯灵随阵眼相对位置移动。
- 飞船结构破坏、撞墙或偏离航道时，灯灵回到上一步或房间入口。
- 后续引入晨风舟时，首个载具房只做水平航线，避免方向和相位同时过载。

可读性要求：

- 阵眼格必须有明确标记。
- 搭乘状态的灯灵视觉应变暗或收缩，表达“不再发光干扰”。
- 航线可在早期房间用水面微光暗示。

## 9. 四叶眠床：镇石与防波堤

`Block / 四叶眠床` 是最稳定的图案，也是后续章节很适合引入的“机关桥梁”。它可以作为压住踏板的镇石，也可以作为阻挡波动的防波堤。

当前第一章三房间版本不强制使用 `Block`，避免在 `Glider` 教学前加入过多图案概念。

房间语法：

```text
残缺四叶眠床 + 可补种碎片 + 重力踏板 + 门或桥
```

用法：

- 压住踏板：只要 `Block` 完整并覆盖踏板区域，门保持开启。
- 防波堤：稳定花床挡住某些生长波，保护接收器或路口。
- 教学意义：告诉玩家“不是所有目标都要移动；不变也是有效状态”。

## 10. 第一章房间类型

| 类型 | 目标 | 适合位置 |
| --- | --- | --- |
| 普通植物房 | 看懂花和蘑菇会随移动、等待、灯灵邻居而变化 | 房间一 |
| 机关植物房 | 补全残缺 `Glider`，让它触碰机关开门 | 房间二 |
| 混合房 | 同时处理普通植物道路阻挡和机关植物触发 | 房间三 |

## 11. 首页与黑板布局

Phase 2 首页建议为：

```text
教程
荧光森林
荧光花园（第一版为占位入口；通关荧光森林后显示解锁回报）
设置
```

关卡界面继续沿用“棋盘 + 黑板”的结构：

| 区域 | 第一章用途 | 后续扩展 |
| --- | --- | --- |
| 棋盘 | 展示房间、灯灵、植物、出口和机关 | 房间地形、动画、机关反馈 |
| 黑板 | 当前目标、规则提示、等待/撤销等操作说明 | 背包、图鉴、花谱、机关植物说明 |

教程保持静态规则说明，不做互动步骤。它可以直接用黑板展示生命游戏四条规则，并配小棋盘例子；动态规则由荧光森林房间承担。

## 12. 房间设计模板

每个房间进入制作前应先写成以下格式：

```text
房间编号：
房间名称：
棋盘尺寸：
入口：
出口：
地形：
初始植物：
机关元素：
玩家目标：
关键教学：
黑板内容：
失败条件：
悔棋需求：
完成奖励：
```

示例：

```text
房间编号：C1-R02
房间名称：残缺的流光种
棋盘尺寸：10 x 10
入口：下方
出口：右上石门
地形：石墙隔开机关与灯灵可行动区域
初始植物：残缺 Glider
机关元素：一枚 Glider 种子，简化机关形状
玩家目标：把种子补回缺口，启动流光种，让它触碰机关
关键教学：机关植物仍遵循生命游戏，但能触发门
黑板内容：补齐并启动流光种，让它飞向机关
失败条件：流光种撞墙或熄灭
悔棋需求：可撤销放置和等待
完成奖励：打开石门，解锁流光种札记
```

## 13. 设计风险

| 风险 | 处理 |
| --- | --- |
| 玩家分不清植物和机关 | 视觉上区分“会生长的花”和“可搬运的月石/核心” |
| 生命游戏预测过难 | 第一章只要求 1 到 4 代内的清晰结果 |
| 房间太像教学题而不是森林 | 用出口、石墙、植物阻挡和回家目标组织空间叙事 |
| `Glider` 对新玩家太抽象 | 房间二先用残缺形状、缺口高亮和占位机关降低理解成本 |
| 黑板信息过多 | 第一章黑板只显示当前目标和一条关键规则，背包/图鉴后续再加 |
| 三房间太短 | 这是第一章最小闭环；成功后再扩成更多房间或支路 |
