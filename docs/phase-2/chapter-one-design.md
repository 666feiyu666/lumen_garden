# Phase 2 第一章设计总纲

## 1. 章节愿景

第一章名义目标是“穿过荧光森林，回到灯漫村”。玩家扮演一只迷路的小灯灵，从黑暗森林遗迹醒来，在连续三个棋盘房间中重新理解世界的规则：植物都会按康威生命游戏自行生长，但普通植物主要构成道路、阻挡和教学环境，机关植物则承担触发、开门和后续谜题工具的功能。

第一章结束时，灯灵不只是找到出口，也完成了身份转变：它从被动迷路的微光，变成能读懂森林植物、修复机关植物、带着第一份花谱回家的小守护者。

## 2. 玩家体验目标

| 体验 | 设计含义 |
| --- | --- |
| 孤独但不绝望 | 房间安静、遗迹空旷，但每次解开机关都会得到温暖回应 |
| 慢速史诗感 | 不靠操作压力制造紧张，而靠跨房间的“我真的走出来了”制造推进感 |
| 规则可读 | 失败应让玩家理解是哪一代、哪一格、哪一个机关判断出了问题 |
| 图案有身体性 | `Glider` 不是抽象形状，而是会飞向机关的机关植物；普通花与蘑菇则是森林本身 |
| 章节有回家感 | 终点不是单纯结算，而是第一次看到灯漫村的灯火，并解锁荧光花园入口 |

## 3. 结构借鉴

Phase 2 借鉴《咩咩启示录》的房间串联方式：玩家不是在一张巨大地图上连续移动，而是在一组短房间之间推进。第一章先采用最清晰的三房间线性结构，每个房间是一张棋盘，有明确入口、出口、房间主题和完成条件。

借鉴点：

| 借鉴内容 | 在《萤光花园》中的转译 |
| --- | --- |
| 短房间节奏 | 每个房间解决一个小谜题或一个机制组合 |
| 路线推进 | 房间之间由遗迹门、桥、洞口或光径连接 |
| 节点选择 | 第一章暂不做分支；后续章节再引入支路、奖励房和资源房 |
| 最终出口 | 主线末端是离开遗迹、进入灯漫村的章节门槛 |

不借鉴内容：

- 不做实时战斗。
- 不以随机战利品作为主要驱动。
- 不把第一章做成可无限重复的 rogue 循环。
- 不让房间数量膨胀到超过手工调校能力。

## 4. 第一章核心循环

```text
进入房间
  阅读棋盘、植物、机关形状与出口
  移动灯灵，观察普通植物如何生长和死亡
  在机关植物房中拾取种子或碎片
  将残缺机关植物补回正确形状
  启动机关植物，让它从休眠形状进入演化
  继续移动或等待，让机关植物触发开门
离开房间
  进入下一张棋盘，最终回到灯漫村
```

第一章的重点不是“每个房间都引入新规则”，而是让玩家先经历三步清晰认知：普通植物会随着灯灵行动自然变化；机关植物在启动前保持休眠，启动后才按同样的生命规则行动；两类植物可以在同一房间里组合。

## 5. 两类植物

荧光森林里的植物分为两种：普通植物和机关植物。它们的底层元素都遵循康威生命游戏规则，差别在于游戏语义和关卡作用。

| 类型 | 示例 | 康威规则 | 游戏作用 |
| --- | --- | --- | --- |
| 普通植物 | 花、蘑菇、普通荧光芽 | 每次灯灵行动都会进行生死判断 | 构成道路阻挡、教学环境、空间节奏和森林生命感 |
| 机关植物 | `Glider`、大炮、未来的载具植物 | 未启动前休眠；启动后按生命规则演化 | 作为谜题工具，触发机关、打开门、发射图案或运输灯灵 |

### 普通植物

普通植物是荧光森林的自然层。它们可以挡住出口或道路，也会因为灯灵靠近、等待或绕行而改变状态。房间一应沿用当前引导模式的思想：玩家不需要先理解全部规则，只要在移动过程中发现“我靠近也会改变植物的生死条件”。

第一章中的普通植物承担三种用途：

| 用途 | 示例 |
| --- | --- |
| 道路阻挡 | 初始植物挡住出口，玩家移动和等待后道路被自然打开 |
| 规则教学 | 灯灵作为邻居影响花、蘑菇的出生和死亡 |
| 空间节奏 | 某些植物周期性出现或消失，暗示等待和观察的重要性 |

### 机关植物

机关植物是荧光森林中的特殊生命结构。它们仍然是植物，仍然遵循康威生命游戏规则，但只有在机关启动后才进入演化；未启动时，它们更像被遗迹封存的休眠形状。例如残缺的 `Glider` 被补齐并启动后会移动，触碰远处机关，从而打开通向下一房间的门。

第一章先用 `Glider` 代表机关植物。大炮等更复杂机关植物可以作为后续章节或花园解锁内容，不在第一章第一版中展开。

机关植物最好满足以下条件：

- 视觉上能和普通花、蘑菇区分。
- 放错位置也能看出为什么不工作。
- 补齐并启动后通过生命游戏演化产生结果。
- 与形状、方向、代数有关，而不是简单“钥匙开门”。
- 允许撤销和重试，避免放错种子带来重复劳动。

## 6. 第一章图案词汇表

| 图案 | 章节名称 | 功能 | 第一章定位 |
| --- | --- | --- | --- |
| 花 / 蘑菇 | 普通植物 | 阻挡、出生、死亡、形成教学环境 | 房间一主角 |
| `Blinker` 或简单振荡图案 | 摇曳花枝 | 两代周期，提示等待与相位 | 可作为房间一或三的普通植物表现 |
| `Glider` | 流光种 | 四代斜向位移，远程触碰机关 | 房间二主角 |
| 大炮 | 待定机关植物 | 周期性产生流光种 | 仅作为未来扩展概念，第一章先不实现 |

第一章不需要引入大型复杂图案。内容密度应来自房间组合，而不是一次塞入太多生命游戏术语。

## 7. 房间迷宫结构

第一章建议采用三房间线性结构，目标是先完成最小可玩章节，而不是一次做成完整大型迷宫。

| 房间 | 主题 | 目标 |
| ---: | --- | --- |
| 01 | 普通植物房 | 花和蘑菇挡住道路与出口；玩家移动灯灵时观察植物生长、死亡，以及灯灵作为邻居的影响 |
| 02 | 机关植物房 | 残缺 `Glider` 位于棋盘中；灯灵拾取或携带一枚种子，将其补回正确位置，启动后让 `Glider` 移动触碰机关 |
| 03 | 混合房 | 同时出现普通植物和机关植物；普通植物随行动变化，机关植物启动后才加入演化 |

房间默认约定：

- 灯灵出现在棋盘下部，出口通常位于上方或右上方。
- 每个房间都使用“棋盘 + 黑板”布局；黑板先承担规则提示和目标说明，后续可扩展为背包、图鉴和花谱。
- 房间二的机关触发物可以先用一个简化形状或占位图标代替正式机关美术。
- 第一章不引入分支、黑水、`LWSS` 或大炮实装；这些作为后续扩展保留。

## 8. 首页与解锁结构

Phase 2 的首页入口建议调整为：

| 入口 | 状态 | 说明 |
| --- | --- | --- |
| 教程 | 默认可进入 | 静态生命游戏规则说明，配合示意图和小棋盘演示 |
| 荧光森林 | 默认可进入 | 第一章主线，包含普通植物房、机关植物房、混合房 |
| 荧光花园 | 默认可见，第一版为占位入口；通关荧光森林后显示解锁回报 | 作为大本营、花谱、自由种植和图鉴入口 |
| 设置 | 默认可进入 | 窗口、音量、语言等 |

教程应是静态规则页，而不是互动关卡。它只负责把生命游戏的基础生死规则讲清楚；动态规则、灯灵影响、普通植物与机关植物的差异，应由玩家在荧光森林房间中学会。

- 有生命的格子，邻居少于 2 会死亡。
- 有生命的格子，邻居为 2 或 3 会存活。
- 有生命的格子，邻居多于 3 会死亡。
- 空格子，邻居恰好为 3 会诞生。
- 灯灵在特定模式下也可以作为一束邻居微光参与判断。

## 9. 灯漫村回归

第一章终点是灯漫村，但 Phase 2 不必一次完成完整经营系统。建议第一章结尾实现“可看见灯漫村，并解锁荧光花园入口”的最小回报。

结尾内容：

- 村口远处有温暖灯火，与遗迹冷色形成对比。
- 灯灵被村中的微光回应，确认“这里是家”。
- 第三个房间出口触发 `end_1.png` 结尾漫画，作为第一章离开荧光森林的情绪回报。
- 玩家把第一章获得的普通植物观察记录和 `Glider` 机关植物收进图鉴。
- 首页中的“荧光花园”入口保持可见；通关前显示占位页，通关后显示回到灯漫村与解锁花园的章节回报。
- 章节标题从“迷路”转为“归灯”，为第二章开启村庄与外出循环留下空间。

## 10. 第一章验收标准

第一章完成时应满足：

- 首页包含“教程、荧光森林、荧光花园、设置”，且荧光花园在第一版中先作为占位入口，通关荧光森林后切换为解锁回报。
- 玩家能从荧光森林入口连续通过三个棋盘房间。
- 房间一只使用普通植物，完成对生长、死亡和灯灵邻居影响的教学。
- 房间二只突出机关植物，用残缺 `Glider` 和一枚可补回的种子完成开门。
- 房间三混合普通植物与机关植物，要求玩家同时处理道路阻挡和机关触发。
- 三个房间默认让灯灵从棋盘下部开始。
- 黑板区域能承担当前目标、规则提示，并为未来背包和图鉴留出位置。
- 第一章结尾具有叙事回报：灯灵离开荧光森林，回到灯漫村，并解锁荧光花园。

## 11. 待确认问题

- 房间二中补回 `Glider` 种子的操作是“拾取后种植”，还是“站到缺口旁按播种”。
- `Glider` 机关触发目标用正式月光接收器，还是第一版先用占位形状。
- 机关植物的“启动”是补齐后自动启动，还是需要玩家按下独立启动键。
- 教程规则页保持静态说明；后续只需要决定视觉表现是否继续使用黑板和示意图。
- 荧光花园解锁后第一版是否需要可玩内容，还是只显示“已回到灯漫村”的章节回报。
