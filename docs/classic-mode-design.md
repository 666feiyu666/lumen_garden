# 《萤光花园》种植花谱模式设计

## 设计结论

经典图案不应成为与灯灵分离的观察模式，而应成为一组以灯灵播种为核心动作的解谜关卡：

- 当前已实现玩法保留为 **自由演化 / 花园谜题**：灯灵移动或停留后立刻触发生长，并作为一束额外微光影响当轮演化。
- 新玩法定位为 **种植花谱 / Planting Bloom**：灯灵在静止花圃中移动，以有限花种布置或补全起始状态；玩家确认后，花圃在纯康威 `B3/S23` 规则下自行生长。
- “经典”的判定不在于灯灵从未参与，而在于目标图案形成后的行为能够脱离灯灵干预，被标准规则验证。

这个方向既保留灯灵作为玩家身份与操作焦点，也让 `Block`、`Blinker`、`Glider` 等经典结构真正成为谜题的结果，而非只能翻看的陈列品。

## 两种模式

现有 UI 仍使用“花园谜题”名称；当双模式导航落地时，可以将其作为“自由演化”的章节名称或副标题。

| 模式 | 灯灵行为 | 演化时机 | 规则 | 主要目标 | 状态 |
| --- | --- | --- | --- | --- | --- |
| 自由演化 / 花园谜题 `Garden Puzzle` | 移动或停留，提供额外微光 | 每次行动后立即生长一代 | `B3/S23` 加灯灵邻居 | 在黎明前达到目标花量 | 已实现 |
| 种植花谱 `Planting Bloom` | 移动并消耗花种播种 | 玩家确认开始后连续演化 | 验证期间为纯 `B3/S23` | 生成或修复具有标准行为的经典花谱 | `P01-P05` 第一册已实现 |

“自由演化”在本项目中表示花圃随灯灵每一步持续变化的现有章节名称，不表示数学意义上的无外力运行；纯规则的自主生长发生在种植花谱的验证阶段。

## 核心原则：先种植，后验证

灯灵可以影响经典图案的形成，但不应在结果验证时继续充当隐形邻居。

以 `Glider / 流光种` 为例，谜题可以要求玩家通过播种补齐一颗能够启程的种子，甚至让它经过一两代才形成完整滑翔机。当玩家按下“开始生长”后，灯灵不再影响邻居计数；只有流光种随后自主完成标准迁移，才算成功。

因此，种植花谱不是直接复用当前灯灵规则，也不是移除灯灵，而是改变灯灵作用的时间位置：

| 阶段 | 灯灵是否可操作 | 灯灵的影响 | 是否进行演化 |
| --- | --- | --- | --- |
| 播种阶段 | 是 | 在合法土格播下有限花种；不作为邻居 | 否 |
| 确认阶段 | 是 | 玩家可重置方案或开始生长 | 否 |
| 生长阶段 | 否，或仅作场外旁观光点 | 不参与邻居计数 | 是，纯 `B3/S23` |
| 验证阶段 | 否 | 根据标准形状、周期或迁移判定成功 | 随目标观察若干代 |

首版不允许演化过程中继续播种。中途播种会形成另一种有趣的高阶玩法，但会模糊经典行为究竟来自初态还是持续外力。

## 基本操作与资源

### 播种阶段

- 玩家移动灯灵至可播种土格；移动本身不消耗花种，也不触发生长。
- 播种使用明确的 `P` 键或“播种”按钮并消耗一枚花种；点击灯灵本身只提示操作，不隐式消费花种。玩家可撤回本次尝试中播下的花种，避免重置带来的重复劳动。
- 花种未全部布置时，`Space` 与 `Enter` 均不会触发播种或验证，而会提示剩余花种；完成播种后两者均可作为冗余确认键放手生长。
- 谜面可以包含预先盛开的固定花芽、禁止播种的石格以及目标轮廓提示。
- 谜题预算以花种数和可播种位置为主，不以灯灵走路步数为主。

### 生长与复盘

- 玩家确认后，花圃自动播放纯规则演化；玩家可暂停或单步查看。
- 失败后可以返回播种阶段保留初始谜面并重新布置。
- 成功时清楚说明验证依据，并进入“成谱演示”：循环回放本次验证的安全片段，允许暂停或从起点重播，而不改写胜负判定。
- 移动花谱的成谱演示不继续向有限花圃边界推进，避免将边缘消散误呈现为图案行为的一部分。

### 常规花圃规格

- 正式第一册的花园谜题与种植花谱均采用 `10 x 10` 花圃，以维持稳定的阅读尺度和视觉表现。
- 花园谜题通过初态、灯灵路线、停留时机、轮数与盛开目标调整难度。
- 种植花谱通过预置花芽、花种预算、可播种区域、轮廓提示和行为目标调整难度。
- 引擎继续支持可变棋盘尺寸，为未来特殊内容保留余地；常规关卡不把扩大棋盘作为升级方式。

## 图案决定谜题类型

经典生命游戏图案不是同质的收藏品。它们的行为类型直接限定了关卡可考验的预测能力：

| 类型 | 示例 | 适合的种植任务 | 设计作用 |
| --- | --- | --- | --- |
| 静物 Still life | `Block`、`Boat`、`Beehive` | 用少量花种让花床稳定下来 | 入门，理解“不再变化”也是目标 |
| 振荡器 Oscillator | `Blinker`、`Toad`、`Beacon` | 补全节律，令花丛按周期呼吸 | 早中期，训练对多代变化的判断 |
| 移动结构 Spaceship | `Glider`、`LWSS` | 种出会自行通过目标区域的流光种或流光舟 | 招牌谜题，把形态、方向与空间目标结合 |
| 大型周期结构 | `Pulsar` | 修复复杂对称节律 | 后期内容，需要更大棋盘和阅读负担 |
| 长期生长种子 | `R-pentomino`、`Acorn` | 触发连锁盛放并观察生命史 | 更适合展览或挑战，不宜作为短局基础题 |

## 第一册种植谜题建议

第一册种植花谱已经以五个 `10 x 10` 关卡完成首轮落地，并支持自动播放、暂停与单步验证。关卡从静物进入周期，再进入两种位移：`Beacon` 为 `Glider` 前的轮廓阅读过渡，`LWSS` 则避免末关只是重复滑翔机方向。

| 顺序 | 谜题名 | 图案目标 | 玩家任务 | 胜利验证 |
| ---: | --- | --- | --- | --- |
| 01 | 四叶眠床 | `Block` | 在残缺花床旁补种极少量花芽 | 连续 3 代保持稳定 |
| 02 | 摇曳花枝 | `Blinker` | 从静止谜面种出会呼吸的花枝 | 第 2 代回到起始相位 |
| 03 | 双灯花台 | `Beacon` | 补齐两簇相对的花床 | 第 2 代回到起始相位 |
| 04 | 流光种 | `Glider` | 直接补全一颗可启程的种子 | 第 4 代平移 `(1, 1)` 到轮廓 |
| 05 | 晨风舟 | `LWSS` | 以两枚花种补齐更宽的轻型流光舟 | 第 4 代水平平移 `(2, 0)` |

避免让关卡退化成照着轮廓填格：

- 不从完全空白花圃开始，而以残缺、干扰或潜伏种子为谜面。
- 首次教学题可以显示目标轮廓，后续只描述行为目标。
- 位移题先要求玩家补齐可识别的标准初态；未来再加入需在演化中自然催生飞船的高阶谜题。
- 预置花芽、有限花种和不可播种格应共同塑造解法，而不是只靠隐藏答案。

## 花谱札记与观察功能

观察仍有价值，但它应是种植谜题的辅助，而不是独立主模式。当前已实现成功后的关内“成谱演示”，独立收藏页仍属于后续内容：

- 谜题开始前可在札记中显示标准英文名、花园别名、类型与目标行为。
- 玩家完成图案后可在当前关内循环观看已验证片段；该循环是演示反馈，不是新的生命规则。
- 玩家首次完成某种结构后，可解锁其标准初态的自由回看页，支持逐代查看和重置。
- 回看页用于帮助玩家理解已发现的结构，不替代播种关卡的交互。

## 规则与实现边界

`P01-P05` 已经落地以下规则边界；后续关卡应在这些边界上扩充，而不是重新混合两种灯灵行为：

| 已实现原型状态 | 后续扩展 |
| --- | --- |
| `patterns.py` 描述 `Block`、`Blinker`、`Beacon`、`Glider`、`LWSS` 及对应种植谜面 | 继续选取可在有限花圃清楚验证的经典行为 |
| `PlantingState` 区分播种阶段和生长验证阶段 | 支持更多关卡目标与进度记录 |
| `grow_once()` 支持带灯灵的自由演化与无额外光源的种植验证 | 保持两种调用路径的规则回归测试 |
| 菜单提供 `P01-P05` 五关选择、显式播种、播放/暂停/单步验证与关内成谱演示 | 后续可加入完成标记与独立花谱回看 |
| `Block` 胜利条件检查初态补全及连续稳定 | 新增周期、形状平移和目标区域到达判定 |

推荐保持以下纯演化职责：

```python
grow_once(blooms, width, height, extra_lights=frozenset()) -> set[Point]
```

- 自由演化 / 花园谜题在每次行动后传入灯灵位置。
- 种植花谱在播种阶段不调用演化；开始生长后始终传入空光源。
- 图案验证使用花芽坐标集合与代数，不依赖像素绘制结果。

首版仍采用有限棋盘且不环绕。涉及 `Glider` 的关卡必须留足移动边距，或在其触及边缘之前完成验证。


种植花谱界面的重点信息：

- 左侧花圃清楚区分预置花芽、可播种格、禁止播种格、目标轮廓与灯灵。
- 若灯灵仍站在刚播下的花芽上，花芽保持可见，并以金色定位标记表示灯灵所在格，避免补种反馈被覆盖。
- 右侧“花谱札记”显示花种余额、当前阶段、目标图案类型和验证条件。
- 播种阶段突出“`P` 播种、`X` 撤回、`Space` / `Enter` 确认、尚未生长、花种必须用完”；花圃内鼠标只移动到相邻格；确认后切换为代数、行为进度和重置入口；成功后显示成谱演示状态。
- 文案保持花种、萌芽、花谱、流光等园艺语汇，同时保留标准英文图案名称和 `B3/S23` 说明。

## 验收标准

### 规则与谜题

- 当前自由演化谜题和教程仍保持现有灯灵邻居规则；正式五关已统一为 `10 x 10` 并以参考路径回归。
- 种植阶段移动灯灵不会触发生长；`P`/按钮播种会正确消耗及撤回花种，花圃内鼠标点击不会隐式播种或开始验证。
- `Space` 与 `Enter` 在花种未用完时都只提供提醒，在花种齐备后都能启动验证。
- 花种未用完时不能启动生长验证，提示不会改变布局、余额或代数。
- 种植谜题开始生长后，灯灵不计入邻居，且玩家不能中途改变验证中的花芽布局。
- `Block` 目标能验证连续稳定。
- `Blinker` 目标能验证周期 2。
- `Beacon` 目标能验证周期 2，并作为更宽轮廓的阅读过渡。
- `Glider` 目标能验证在充足空间内第 4 代相对标准形状平移 `(1, 1)`。
- `LWSS` 目标能验证在充足空间内第 4 代水平平移 `(2, 0)`。
- 成功后的成谱演示仅回放已验证片段，不让 `Glider` 或 `LWSS` 因展示推进而撞到有限花圃边缘。

### 体验

- 玩家能从菜单区分当前的自由演化玩法和新的种植花谱玩法。
- 首个种植题不是被动观看，而要求玩家消耗花种做出选择后启动生长。
- 结果区域说明成功来自哪一种标准行为，并保持花圃可见以继续观看成谱演示。
- 关内成谱演示已可展示经典图案行为；独立札记收藏页仍不阻塞谜题交付。

## 风险与设计决定

| 风险 | 当前决定 |
| --- | --- |
| 灯灵持续干预导致经典行为含糊 | 灯灵只在播种阶段改变初态；验证阶段不参与邻居计算 |
| 种植变成照轮廓填空 | 使用残缺谜面、少量花种和延迟生成目标，逐步撤掉轮廓提示 |
| 移动成本拖慢布置体验 | 播种阶段移动不触发生长，首版不以移动步数作为主要预算 |
| 允许中途播种后规则难以教学 | 首版禁止；将其保留为后期特殊挑战可能性 |
| `Glider` 撞到有限棋盘边缘 | 提供足够留白并在碰撞边界前验证 |
| 成功后继续播放使移动图案撞边 | 成谱演示循环回放验证安全片段，不把循环重置冒充规则演化 |

## 待试玩确认的问题

- 首批谜题是否需要铲除预置花芽；首版更稳妥的范围是仅允许播种与撤回玩家所种的花。
- `Beacon` 到 `Glider`、`LWSS` 的实际难度过渡是否清楚，仍需试玩观察。
- `P` 显式播种、`Space` / `Enter` 冗余确认与成谱安全片段循环是否让首次玩家更容易理解操作和动态目标。
- 完成图案后解锁标准花谱回看页，是否足以承担规则教学与收集欲。

2. 已完成：以 `P01 Block / 四叶眠床` 定义播种、撤回、开始生长、验证和重置循环。
3. 已完成：将正式花园第一册与 `P01` 统一为 `10 x 10`，并为种植验证增加自动播放、暂停与单步反馈。
4. 已完成：制作 `P02-P05`，以周期 2 与两类飞船位移形成种植花谱第一册。
5. 已完成：以 `P` 建立显式播种，以 `Space` / `Enter` 提供齐备后的冗余确认，阻止未完成布局提交，并在成功后提供已验证片段的成谱循环演示。
6. 下一步试玩：校准双模式五关的实际难度梯度，并验证玩家是否理解“先播种，后放手”及动态演示。
