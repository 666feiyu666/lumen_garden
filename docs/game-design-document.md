# 《萤光花园 / Lumen Garden》游戏设计文档

## 1. 项目概述

| 项目项 | 内容 |
| --- | --- |
| 名称 | 萤光花园 / Lumen Garden |
| 类型 | 温暖、短局、回合制规则解谜 |
| 平台 | PC，Python + pygame 原型 |
| 当前内容 | 月夜入门 + 统一 `10 x 10` 的五则正式花园谜题 + `P01-P05` 种植花谱第一册 |
| 规则骨架 | 花园谜题为灯灵作为移动邻居的康威变体；种植花谱以纯规则验证经典图案 |
| 当前方向 | 以月夜花圃替代旧“变异细胞/实验室”包装 |

方向决策与保留候选主题见 [戏剧方向决策](narrative-directions.md)。

## 2. 核心体验

玩家是一枚可移动的金色灯灵。夜色中的绿色花芽会依照相邻微光数量盛开或熄灭。灯灵的照料方式因章节而不同：

- 当前已实现的自由演化 / 花园谜题中，玩家不直接种花，而是选择灯灵的位置，在黎明倒计时结束前影响花圃并达到目标盛开数量。
- 已实现的种植花谱中，玩家以有限花种布置起始花圃，然后放手让经典图案依纯规则展现稳定、呼吸或迁移。

设计支柱：

| 支柱 | 说明 |
| --- | --- |
| 温柔而可推演 | 规则产生明确后果，但反馈不恐怖、不惩罚式 |
| 少操作高影响 | 移动、停留或有限播种足以形成丰富解法 |
| 光的引导感 | 灯灵提供关键微光或种下起始花芽，玩家感到自己在照料而非处置 |
| 谜题册结构 | 内容以有名字的谜面组织，而非简单的难度关卡表 |

## 3. 当前自由演化玩法规则

### 3.1 对象

| 对象 | 玩法作用 | 表现 |
| --- | --- | --- |
| 金色灯灵 | 玩家控制，不会熄灭；贡献一位相邻微光；不计入目标 | 暖金发光小精灵 |
| 绿色花芽 | 会依生长规则盛开或熄灭；计入目标 | 小型五瓣发光花 |
| 花圃土格 | 限定移动与邻近关系 | 柔和石边土格 |

### 3.2 每步流程

1. 玩家移动灯灵一格，或选择停留。
2. 若移动越界或落点已有花芽，动作拒绝且不消耗步数。
3. 所有花芽同时生长一轮。
4. 黎明倒计时减少，HUD 更新盛开数量。
5. 步数归零时检查盛开目标。

### 3.3 生长规则

| 状态 | 相邻微光数量 | 下一轮 |
| --- | ---: | --- |
| 盛开花芽 | 2 或 3 | 保持盛开 |
| 盛开花芽 | 小于 2 或大于 3 | 熄灭 |
| 空土格 | 恰好 3 | 萌出花芽 |

灯灵参与邻近微光计数，但不会成为被统计的花芽。

### 3.4 种植花谱规则（`P01-P05` 已实现）

| 阶段 | 玩家操作 | 灯灵作用 | 花圃规则 |
| --- | --- | --- | --- |
| 播种 | 移动、以 `P`/按钮有限播种、撤回本次播种 | 播下初始花芽，不提供邻近微光 | 静止，不演化；花种未用完不可提交 |
| 生长验证 | 播种齐备后以 `Space`/`Enter`/按钮确认开始、暂停/单步、重置 | 退到旁观位置，不影响花圃 | 纯 `B3/S23` |
| 成谱演示 | 暂停/继续、片段重播、重置 | 不再介入 | 回放已验证帧，不改写胜负判定 |

种植花谱以经典结构的行为为目标，不以最终花芽数量代替判断：静物验证稳定，振荡器验证周期，移动结构验证相对位置变化。

## 4. 内容结构

### 4.1 月夜入门

`00 GUIDE / 月夜入门` 已实现，以互动分镜解释熄灭、相伴、萌芽和灯灵引路，并以一则两步小谜题收束教学。

### 4.2 第一册花园谜题

| 编号 | 名称 | 花圃 | 可用步数 | 盛开目标 | 情绪主题 |
| ---: | --- | ---: | ---: | ---: | --- |
| 01 | AWAKEN 初光 | 10 x 10 | 3 | 3 | 第一次用灯灵催生新芽 |
| 02 | SPROUT 新芽 | 10 x 10 | 5 | 10 | 连续靠近并延续花光 |
| 03 | CHORUS 合鸣 | 10 x 10 | 6 | 19 | 两簇花群相遇与停留时机 |
| 04 | MOONBED 月圃 | 10 x 10 | 7 | 19 | 主动停驻形成的照料取舍 |
| 05 | DAWN 黎明 | 10 x 10 | 9 | 25 | 多步路线的综合收束 |

正式第一册不沿用原型时期通过扩大棋盘制造复杂度的五套布局。常规谜题统一采用 `10 x 10`，通过初态、灯灵路径、停留时机、行动轮数和盛开目标建立难度梯度。底层尺寸能力仍保留给未来特殊谜题；当前参考路径用于功能回归，实际难度仍需试玩校准。

### 4.3 双模式发展方向

现有内容应保留为 **自由演化 / 花园谜题 / Garden Puzzle**：玩家每次移动或停留灯灵后，立刻影响花圃生长，并以盛开数量为解谜目标。当前界面名称无需在新增模式前提前改动。

已经加入 **种植花谱 / Planting Bloom** 第一册五关：`Block` 教授静物稳定，`Blinker` 与 `Beacon` 展示周期，`Glider` 与 `LWSS` 分别验证斜向和横向迁移。五关使用同样的 `10 x 10` 常规花圃；玩家以 `P` 明确播种，花圃内鼠标仅选择相邻移动，花种用完后可按 `Space` 或 `Enter` 确认，花圃自动播放纯 `B3/S23` 演化。验证成功后可以循环观看已验证片段，不让有限棋盘上的移动花谱继续撞向边界。

种植花谱中的灯灵依然影响结果，但影响发生在播种阶段，而不是验证阶段持续提供额外邻居。`P01-P05` 都要求玩家播种并启动演化；已完成图案的观察与札记页仍可后续补充。详细方案及开发门槛见 [种植花谱模式设计](classic-mode-design.md)。

## 5. UI 与视觉

### 5.1 已落地的 UI 转向

- 窗口与标题改为《萤光花园 / Lumen Garden》。
- 启动界面改为简洁主菜单，仅提供“花园引导”“种植花谱”“设置”入口；内容卡片下沉到模式选择页。
- 设置以弹窗形式提供三档窗口尺寸、音量调整与静音切换。
- 主菜单、模式选择、设置及两套玩法的主要行动均支持鼠标点击。
- 游戏 HUD 改为盛开花芽、盛开目标、黎明倒计时和生长轮次。
- 棋盘使用月夜土格；普通对象绘制为绿色花芽；玩家绘制为暖金灯灵。
- 结算语汇改为“花园盛放”与“月夜散场”；种植花谱成功后以不遮挡花圃的“成谱演示”继续展示行为。

### 5.2 后续视觉深化

| 元素 | 方向 |
| --- | --- |
| 背景 | 靛蓝月色、柔和草地层次、极少量萤火 |
| 花圃 | 土格和圆石边缘，不做玻璃或仪器面板 |
| 灯灵 | 温暖金色、简洁光环、移动拖光 |
| 花芽 | 小巧花瓣轮廓，盛放/熄灭状态清晰 |
| 面板 | 园艺手记或木玻璃卡片，轻量、留白充足 |

概念图生成提示见 [美术概念图 Prompts](art-prompts.md)。

## 6. 音频方向

| 事件 | 声音建议 |
| --- | --- |
| 主菜单 | 夜风、虫鸣、稀疏铃音 |
| 灯灵移动 | 柔和木铃或微小闪光音 |
| 新芽盛放 | 清亮的短音，不刺耳 |
| 花芽熄灭 | 轻柔的落叶/气息音 |
| 谜题解开 | 暖色和弦与少量晨光感 |
| 月夜散场 | 安静收束，鼓励重试而非惩罚 |

## 7. 技术结构

| 模块 | 当前责任 | 说明 |
| --- | --- | --- |
| `Lumen_Garden/model.py` | 生长计算、自由演化状态、种植验证状态 | 已抽取 `grow_once()` 支持两套玩法 |
| `Lumen_Garden/puzzles.py` | 五则谜题布局与已知解法 | 导出 `PUZZLES` |

| `Lumen_Garden/app.py` | 场景、输入、花园 UI 绘制 | 已加入显式播种、验证播放及成功后的成谱演示循环 |

| `Lumen_Garden/patterns.py` | 经典目标、种植谜面、花种预算与验证条件 | 已定义五类标准图案及 `P01-P05` |
